## 1. תקציר מנהלים

OctoBankX היא פלטפורמת תוכנה לאוטומציה מלאה של הורדת דפי חשבון בנקאיים. המערכת מתחברת לשרתי ה-SFTP של הבנקים, מורידה קבצי חשבון יומיים עבור כל חשבון רשום, ומנהלת מעקב מלא אחר מצב כל הורדה — בזמן אמת.

המערכת מציעה:

- **ממשק ווב לשולחן עבודה** לניהול מלא של הבנקים, החשבונות, המשימות וההגדרות
- **ממשק נייד מותאם** לצפייה ושליטה מכל מכשיר
- **ממשק API** לאינטגרציה עם מערכות חיצוניות
- **תמיכה בעברית ובאנגלית** עם פריסה RTL מלאה

## 2. הצורך העסקי

### הבעיה הקיימת

ארגונים המנהלים חשבונות בנק מרובים — לעיתים אצל מספר בנקים שונים — מתמודדים עם האתגרים הבאים מדי יום:

| אתגר | תיאור |
|------|--------|
| **עבודה ידנית וחוזרת** | איש כספים נכנס מדי יום לאתרי הבנקים, מוריד קבצים ושומר אותם בתיקיות ידנית |
| **חוסר עקביות** | שמות קבצים, מבנה תיקיות ותהליכי שמירה משתנים בין עובדים |
| **היעדר ביקורת** | אין מסלול ביקורת (audit trail) — לא ברור מה הורד, מתי, ועל ידי מי |
| **כשלים שעוברים מתחת לרדאר** | הורדה שנכשלה עלולה לא להתגלות עד שהנתונים יחסרו בעיבוד החשבונאי |
| **קשיי מדרגיות** | הוספת חשבון חדש מחייבת הוראות עבודה חדשות ותיאום בין אנשי צוות |

## 3. תיאור המערכת

### מודולים ראשיים

#### מודול ניהול בנקים וחשבונות
המשתמש מגדיר את הבנקים שעמם עובד הארגון — כל בנק עם פרטי ה-SFTP שלו (כתובת שרת, פורט, נתיב תיקייה). לאחר מכן מרשם חשבונות בנק, כל חשבון עם אישורי SFTP ייחודיים (שם משתמש וסיסמה שסיפק הבנק).

#### מודול הורדה אוטומטית
ב-06:00 בכל יום (ניתן לשינוי), המערכת מריצה אוטומטית משימת הורדה עבור כל החשבונות הרשומים:

```
לכל חשבון:
  1. יצירת רשומת "ממתין" (pending)
  2. התחברות לשרת SFTP של הבנק
  3. הורדת קובץ דף החשבון לתאריך היום
  4. שמירת הקובץ בנתיב מסודר:
     /downloads/{שם_בנק}/{מספר_חשבון}/{YYYYMMDD}_{מספר_חשבון}.csv
  5. עדכון הרשומה: הצלחה / כישלון + הודעת שגיאה
```

#### מודול ניטור וביקורת
- **דף משימות (Jobs):** מצב כל הורדה בזמן אמת — בריצה / הצליח / נכשל / ממתין
- **דף יומן (Log):** היסטוריה מלאה 90 יום אחורה עם חיפוש וסינון
- **דף קריאות API:** רישום כל קריאה ל-API החיצוני לצרכי אבחון

#### מודול הגדרות
ממשק נוח לשינוי כל פרמטרי המערכת: תיקיית יעד, לוח זמנים, מספר ניסיונות חוזרים, זמן קצוב וכתובת מייל להתראות.

#### REST API
כל הפונקציות חשופות גם כ-API מתועד לאינטגרציה עם מערכות חשבונאות, ERP ו-BI.

### פירוט הטבלאות

#### טבלת `banks` — בנקים
מאחסנת את פרטי ה-SFTP של כל בנק. חשבונות מרובים יכולים להיות קשורים לאותו בנק.

| עמודה | סוג | אילוצים | תיאור |
|-------|-----|---------|--------|
| `id` | INTEGER | PRIMARY KEY | מזהה ייחודי |
| `name` | STRING | NOT NULL, UNIQUE | שם הבנק לתצוגה |
| `sftp_host` | STRING | NOT NULL | שרת ה-SFTP (למשל `sftp.leumi.co.il`) |
| `sftp_port` | INTEGER | DEFAULT 22 | פורט ה-SFTP |
| `sftp_remote_path` | STRING | DEFAULT '/' | נתיב התיקייה בשרת המרוחק |
| `created_at` | DATETIME | — | נוצר אוטומטית |
| `updated_at` | DATETIME | — | עודכן אוטומטית |

#### טבלת `accounts` — חשבונות בנק
חשבונות בנק עם אישורי SFTP ייחודיים לכל חשבון, ופרטים פיננסיים בסיסיים.

| עמודה | סוג | אילוצים | תיאור |
|-------|-----|---------|--------|
| `id` | INTEGER | PRIMARY KEY | מזהה ייחודי |
| `name` | STRING | NOT NULL | שם תצוגה (למשל "ICL Group EUR Treasury") |
| `account_no` | STRING | NOT NULL, UNIQUE | מספר החשבון |
| `bank_id` | INTEGER | FK → banks.id, NOT NULL | הבנק שאליו שייך החשבון |
| `branch` | STRING | — | מספר סניף |
| `currency` | STRING | DEFAULT 'USD' | קוד מטבע (ILS, USD, EUR...) |
| `balance` | FLOAT | DEFAULT 0.0 | יתרה נוכחית |
| `balance_date` | DATE | — | תאריך עדכון היתרה |
| `sftp_username` | STRING | — | שם משתמש SFTP לחשבון זה |
| `sftp_password` | STRING | — | סיסמת SFTP לחשבון זה |
| `created_at` | DATETIME | — | נוצר אוטומטית |
| `updated_at` | DATETIME | — | עודכן אוטומטית |

#### טבלת `downloads` — הורדות
לב המערכת. כל שורה מייצגת ניסיון הורדה יחיד — חשבון אחד, יום אחד.

| עמודה | סוג | אילוצים | תיאור |
|-------|-----|---------|--------|
| `id` | INTEGER | PRIMARY KEY | מזהה ייחודי |
| `account_id` | INTEGER | FK → accounts.id, NOT NULL | החשבון שהורד |
| `bank_id` | INTEGER | FK → banks.id, NOT NULL | הבנק (denormalized לביצועים) |
| `date` | DATE | NOT NULL | תאריך דף החשבון |
| `status` | STRING | DEFAULT 'pending' | `pending` / `running` / `success` / `failed` |
| `error_message` | STRING | — | הודעת שגיאה אם נכשל |
| `file_path` | STRING | — | נתיב הקובץ המקומי אם הצליח |
| `started_at` | DATETIME | — | תחילת ניסיון ה-SFTP |
| `completed_at` | DATETIME | — | סיום (הצלחה או כישלון) |
| `created_at` | DATETIME | — | נוצר אוטומטית |

#### טבלת `settings` — הגדרות
מאגר Key-Value לכל הגדרות המערכת — ניתן לעדכון דרך ממשק המשתמש.

| עמודה | סוג | ערך ברירת מחדל | תיאור |
|-------|-----|----------------|--------|
| `alert_email` | STRING | `ops@octobankx.local` | נמען התראות כישלון |
| `download_dir` | STRING | `/tmp/octobankx/downloads` | תיקיית יעד לקבצים |
| `job_schedule` | STRING | `0 6 * * *` | ביטוי cron להרצה אוטומטית |
| `max_retries` | INTEGER | `3` | ניסיונות חוזרים מקסימאליים |
| `notify_on_fail` | BOOLEAN | `true` | שלח מייל בכישלון |
| `retention_days` | INTEGER | `90` | ימי שמירת היסטוריה |
| `sftp_timeout` | INTEGER | `30` | זמן קצוב לחיבור SFTP (שניות) |

#### טבלת `api_calls` — קריאות API
מסלול ביקורת לכל קריאה ל-REST API החיצוני. משמש לאבחון ולניטור.

| עמודה | סוג | תיאור |
|-------|-----|--------|
| `id` | INTEGER | מזהה ייחודי |
| `http_method` | STRING | `GET`, `POST`, `PATCH` |
| `endpoint` | STRING | נתיב ה-URL (למשל `/api/v1/downloads`) |
| `host` | STRING | כתובת IP / שם המארח של הקורא |
| `account_id` | INTEGER | החשבון הקשור (nullable) |
| `status` | STRING | `success` / `failed` |
| `http_status` | INTEGER | קוד תגובת HTTP (200, 422, 404...) |
| `duration_ms` | INTEGER | זמן עיבוד הבקשה במילישניות |
| `error_message` | STRING | פרטי שגיאה |
| `created_at` | DATETIME | חותמת זמן הקריאה |
