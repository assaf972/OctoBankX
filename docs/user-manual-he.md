# OctoBankX — מדריך למשתמש

OctoBankX הוא מערכת לניהול הורדות דפי חשבון בנקאיים. המערכת מתחברת לבנקים שלך באמצעות SFTP, מורידה אוטומטית דפי חשבון יומיים עבור כל החשבונות המוגדרים, ומספקת מעקב מלא אחר כל ניסיון הורדה. כלול ממשק ווב לשולחן עבודה וממשק מותאם לנייד.

## 1. לוח הבקרה

לוח הבקרה הוא דף הבית של OctoBankX. הוא מספק סקירה כללית של הפעילות האחרונה ומאפשר רישום חשבונות בנק חדשים.

![לוח הבקרה](../screenshots/01_home.png)

### 10 ההורדות האחרונות

הפאנל השמאלי מציג את 10 ניסיונות ההורדה האחרונים בכל החשבונות. כל שורה מציגה:

| עמודה | תיאור |
|-------|--------|
| **תאריך** | תאריך ושעת ניסיון ההורדה |
| **חשבון** | שם החשבון ומזהה |
| **בנק** | הבנק אליו שייך החשבון |
| **סטטוס** | `success` הצלחה, `failed` כישלון, `running` בריצה, `pending` ממתין |
| **שגיאה** | הודעת שגיאה אם ההורדה נכשלה |

תגי הסטטוס מקודדים בצבע: ירוק להצלחה, אדום לכישלון, כחול לריצה, ואפור לממתין.

### טבלת חשבונות

הפאנל התחתון מפרט את כל החשבונות הרשומים עם הבנק שלהם, מספר החשבון, מטבע, יתרה ותאריך עדכון אחרון של היתרה.

### טופס חשבון חדש

השתמש בטופס שמימין כדי לרשום חשבון בנק חדש:

1. **מספר חשבון** — הזן את מספר החשבון המלא
2. **בנק** — בחר את הבנק מהרשימה הנפתחת (יש לרשום את הבנק תחילה; ראה [בנקים](#2-בנקים))
3. **מטבע** — קוד מטבע בן שלוש אותיות (למשל `EUR`, `USD`, `ILS`)
4. **יתרה** — יתרת פתיחה נוכחית
5. **תאריך יתרה** — התאריך שבו נרשמה היתרה
6. **שם משתמש SFTP** — שם משתמש ל-SFTP שסופק על ידי הבנק
7. **סיסמת SFTP** — סיסמת ה-SFTP לחשבון זה

לחץ על **Create Account** כדי לשמור.

### הפעלת הורדות עכשיו

הכפתור **Run Downloads Now** (למעלה מימין) מפעיל משימת הורדה מיידית לכל החשבונות עבור היום, מבלי להמתין להרצה המתוזמנת.

## 2. בנקים

דף הבנקים מנהל את פרטי חיבור ה-SFTP לכל בנק שאתה עובד איתו.

![בנקים](../screenshots/02_banks.png)

### בנקים רשומים

הפאנל הימני מפרט את כל הבנקים המוגדרים ומציג:

| עמודה | תיאור |
|-------|--------|
| **שם** | שם הבנק לתצוגה |
| **SFTP Host** | שם המארח של שרת ה-SFTP של הבנק |
| **פורט** | פורט SFTP (בדרך כלל `22`) |
| **נתיב מרוחק** | התיקייה בשרת ה-SFTP שבה מופקדים דפי החשבון |
| **חשבונות** | מספר החשבונות הרשומים תחת בנק זה |

### הוספת בנק חדש

מלא את טופס **New Bank** מהצד השמאלי:

1. **שם** — תווית מזהה לבנק (למשל `Bank Leumi`)
2. **SFTP Host** — שם המארח של השרת (למשל `sftp.leumi.co.il`)
3. **פורט SFTP** — ברירת מחדל `22`; שנה אם הבנק שלך משתמש בפורט לא סטנדרטי
4. **נתיב מרוחק** — נתיב התיקייה בשרת ה-SFTP (למשל `/statements`)

לחץ על **Add Bank** לשמירה. הבנק החדש יופיע מיד ברשימת הבנקים הרשומים ויהיה זמין בתפריט הנפתח לרישום חשבונות.

## 3. משימות

דף המשימות מציג כל משימת הורדה שבוצעה או נמצאת בתור, עם אפשרויות סינון שיעזרו לך למצוא את מה שאתה צריך.

![משימות](../screenshots/03_jobs.png)

### מסננים

בראש הדף ניתן לסנן את רשימת המשימות לפי:

- **סטטוס** — `all` הכל, `success` הצלחה, `failed` כישלון, `running` בריצה, `pending` ממתין
- **תאריך** — צמצם תוצאות לתאריך ספציפי

### רשימת משימות

כל שורה בטבלה מייצגת ניסיון הורדה יחיד לחשבון אחד ביום אחד:

| עמודה | תיאור |
|-------|--------|
| **תאריך** | התאריך שבו רצה המשימה |
| **חשבון** | שם החשבון |
| **בנק** | הבנק שאליו שייך החשבון |
| **סטטוס** | תוצאת הניסיון |
| **שגיאה** | פרטי השגיאה אם המשימה נכשלה |
| **ניסיונות חוזרים** | כמה ניסיונות חוזרים בוצעו |
| **קובץ** | שם הקובץ שהורד, אם הצליח |

### הפעלת משימה ידנית

השתמש בכפתור **Run Downloads Now** כדי להפעיל הורדה מיידית לכל החשבונות עבור תאריך היום. הדף יתרענן ורשומות `running` או `pending` חדשות יופיעו.

## 4. יומן

דף היומן הוא היסטוריית ההורדות המלאה — מסלול ביקורת שלם של כל הורדת דף חשבון שנוסתה אי פעם.

![יומן](../screenshots/04_log.png)

### מסננים

היומן תומך במספר מסננים שניתן לשלב:

- **חשבון** — סנן לפי חשבון ספציפי
- **בנק** — סנן לפי בנק
- **סטטוס** — `all` הכל, `success`, `failed`, `running`, `pending`
- **מתאריך / עד תאריך** — בורר טווח תאריכים

### רשומות יומן

כל רשומה מציגה את אותם שדות כמו בדף המשימות. היומן מסודר מהחדש לישן ושומר רשומות בהתאם להגדרת `retention_days` (ברירת מחדל: 90 יום).

> **טיפ:** השתמש בדף היומן לצרכי ביקורת ופתרון בעיות — דף המשימות מתמקד בפעילות היום, בעוד שהיומן מכסה את ההיסטוריה המלאה.

## 5. קריאות API

דף קריאות ה-API מתעד כל קריאה יוצאת שביצעה OctoBankX לשירותים חיצוניים, שימושי לאבחון בעיות שילוב.

![קריאות API](../screenshots/05_api_calls.png)

### מסננים

- **סטטוס** — סנן לפי קוד תגובת HTTP או תוצאה
- **תאריך** — סנן לתאריך ספציפי

### רשומות קריאות API

| עמודה | תיאור |
|-------|--------|
| **חותמת זמן** | מתי בוצעה הקריאה |
| **נקודת קצה** | ה-URL או השירות שנקרא |
| **שיטה** | שיטת HTTP (`GET`, `POST` וכו') |
| **סטטוס** | קוד תגובת HTTP |
| **משך** | כמה זמן הקריאה נמשכה (מילישניות) |
| **שגיאה** | פרטי שגיאה אם הקריאה נכשלה |

דף זה עוזר למנהלי מערכת לאבחן בעיות קישוריות עם נקודות SFTP של הבנק או כל שירות משולב אחר.

## 6. הגדרות

דף ההגדרות שולט בהתנהגות המערכת הכללית של OctoBankX.

![הגדרות](../screenshots/06_settings.png)

### שדות תצורה

| הגדרה | תיאור | ברירת מחדל |
|-------|--------|------------|
| **alert_email** | כתובת מייל שמקבלת התראות כישלון | `ops@octobankx.local` |
| **download_dir** | תיקייה מקומית שבה נשמרים דפי החשבון שהורדו | `/tmp/octobankx/downloads` |
| **job_schedule** | ביטוי cron למשימת ההורדה היומית האוטומטית | `0 6 * * *` (6:00 בבוקר) |
| **max_retries** | מספר מקסימלי של ניסיונות חוזרים ל-SFTP לחשבון ליום | `3` |
| **notify_on_fail** | שליחת מייל התראה כאשר הורדה נכשלת (`true`/`false`) | `true` |
| **retention_days** | מספר הימים לשמירת היסטוריית הורדות במסד הנתונים | `90` |
| **sftp_timeout** | זמן קצוב לחיבור SFTP בשניות | `30` |

לחץ על **Save Settings** להחלת השינויים. שינויים ב-`job_schedule` ייכנסו לתוקף בהרצה המתוזמנת הבאה.

> **שים לב:** `download_dir` חייב להיות ניתן לכתיבה על ידי תהליך האפליקציה. ודא שהתיקייה קיימת לפני השמירה.

## 7. ממשק נייד

OctoBankX כולל ממשק ווב נייד מותאם לחלוטין הנגיש בכתובת `/mobile/`. כל אותם הדפים זמינים דרך סרגל ניווט תחתון.

### לוח בקרה נייד

![לוח בקרה נייד](../screenshots/07_mobile_home.png)

לוח הבקרה הנייד מציג מונים מסכמים בראש (סך החשבונות, פעילים היום, כשלונות) ולאחר מכן רשימה גלילה של הורדות אחרונות — כל אחת מוצגת ככרטיס עם שם החשבון, הבנק ותג הסטטוס.

הכפתור **+ New Account** בתחתית פותח טופס מובנה לרישום חשבון חדש.

### בנקים בנייד

![בנקים בנייד](../screenshots/08_mobile_banks.png)

הבנקים מוצגים כקלפים, כל אחד מציג את מארח ה-SFTP, הפורט, הנתיב המרוחק ומספר החשבונות. הקש על **+ New Bank** בתחתית להרחבת טופס הרישום.

### משימות בנייד

![משימות בנייד](../screenshots/09_mobile_jobs.png)

רשימת המשימות מותאמת לנייד עם פריסת קלפים קומפקטית. מסנני הסטטוס מופיעים כשבבים (chips) לחיצים בראש. כל קלף מציג את החשבון, הבנק, התאריך, הסטטוס וכל הודעת שגיאה.

### יומן בנייד

![יומן בנייד](../screenshots/10_mobile_log.png)

היומן בנייד משתמש באותה פריסת קלפים כמו המשימות. פקדי הסינון נגישים דרך סרגל סינון מתקפל בראש.

### קריאות API בנייד

![קריאות API בנייד](../screenshots/11_mobile_api_calls.png)

היסטוריית קריאות ה-API מוצגת כרשימת קלפים גלילה, עם חותמת זמן, נקודת קצה, קוד סטטוס ומשך גלויים במבט ראשון.

### הגדרות בנייד

![הגדרות בנייד](../screenshots/12_mobile_settings.png)

שדות ההגדרות מוצגים כטופס בעיצוב כהה. כל אפשרויות התצורה זמינות. קישור **Switch to Desktop Version** בתחתית מנווט לממשק שולחן העבודה המלא.

### מתג שפה

בכל דפי הנייד, הקש על **EN** או **עב** בפינה הימנית העליונה כדי לעבור בין ממשק בשפה האנגלית לעברית. ממשק העברית משתמש בפריסה מימין לשמאל (RTL).

## 8. מדריך הגדרות

| הגדרה | סוג | תיאור |
|-------|-----|--------|
| `alert_email` | מחרוזת (אימייל) | נמען לכל מיילי התראות המערכת |
| `download_dir` | מחרוזת (נתיב) | נתיב מוחלט לתיקיית דפי החשבון המקומית |
| `job_schedule` | מחרוזת (cron) | ביטוי cron סטנדרטי בן 5 שדות השולט בלוח הזמנים של ההורדה האוטומטית |
| `max_retries` | מספר שלם | כמה פעמים המערכת מנסה שוב חיבור SFTP שנכשל לפני שמסמנת את המשימה כ-`failed` |
| `notify_on_fail` | בוליאני | כאשר `true`, נשלח מייל ל-`alert_email` עבור כל הורדה שנכשלה |
| `retention_days` | מספר שלם | רשומות יומן ישנות יותר ממספר זה של ימים נמחקות אוטומטית |
| `sftp_timeout` | מספר שלם | שניות להמתנה לפני ביטול ניסיון חיבור SFTP |
